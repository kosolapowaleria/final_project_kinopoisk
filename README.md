# Автоматизация UI- и API-тестов для веб-приложения «Кинопоиск»

## 1. Описание задачи проекта

Цель проекта — автоматизировать UI‑ и API‑тесты для веб‑приложения [Кинопоиск](https://www.kinopoisk.ru) с фокусом на тестировании функционала поиска. Проект создан на основе [проекта по ручному тестированию](https://kosolapowaleria.yonote.ru/share/a0805293-8397-4ba9-b8dc-d23b70e8fd17).

Автоматизация позволяет:
- сократить время на регрессионное тестирование;
- обеспечить стабильное покрытие ключевых сценариев;
- упростить проверку изменений в функционале поиска.

## 3. Используемые технологии и инструменты

- **Python** — основной язык программирования для написания тестов;
- **Selenium** — библиотека для автоматизации взаимодействия с веб‑браузером;

- **Allure** — инструмент для генерации отчётов о выполнении тестов;

## 4. Форматирование кода

Код форматируется в соответствии с **PEP8** (стиль написания кода на Python):
- используются **docstrings** для документирования методов и функций;
- все шаги теста размечаются с помощью `allure.step` или `with allure.step` для улучшения читаемости отчёта.

## 5. Запуск тестов

Проект поддерживает **три режима запуска тестов**:

### 1. Запуск только UI‑тестов

```bash
pytest test/test_ui.py -v
```

### 2. Запуск только API‑тестов

```bash
pytest test/test_api.py -v
```

### 3. Запуск всех тестов (UI + API)

```bash
pytest -v
```

### Дополнительные опции

* `-v` — подробный вывод (verbose);

* `--alluredir=./allure-results` — генерация отчёта Allure в папке allure-results (указано в pytest.ini).

## Ограничения

### Капча на Кинопоиске

Автотесты **не обрабатывают капчу** на сайте [Кинопоиска.](https://www.kinopoisk.ru/)

**Причина:** капча появляется при обнаружении автоматизированных запросов — это механизм защиты сайта.

**Последствия:**
* Если капча не появилась — тесты проходят успешно.
* Если капча появилась — тест прерывается (обычно с ошибкой таймаута ожидания элемента).


**Рекомендации по запуску:**
1. Запускайте тесты в периоды низкой нагрузки на сайт (например, ночью или ранним утром).
2. Добавляйте задержки между тестами (`time.sleep(2–3)`), чтобы имитировать «человеческое» поведение.
3. При появлении капчи:
   * вручную решите её в браузере;
   * перезапустите тесты.

**Долгосрочные решения (для повышения стабильности):**
* получение доступа к тестовой среде Кинопоиска без капчи;
* переход на API‑тестирование для проверки данных без UI‑взаимодействия;
* использование мок‑сервера для изоляции тестов от внешних факторов.

## 6. Настройка окружения

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Настройте переменные окружения (при необходимости):
* скопируйте файл `.env.example` в `.env`;

* заполните необходимые переменные (например, `KINO_API_TOKEN`).

3. Получите токен API:
* подробная инструкция доступна в [документации API;]('https://poiskkino.dev/documentation#description/introduction')
* добавьте токен в файл `.env` в виде: `KINO_API_TOKEN=ваш_токен_здесь`;
* если токен не задан в `.env`, система возьмёт его из поля `"token"` в файле `test_data.json`.

---

## 7. Тестовые данные 

Файл `test_data.json` содержит ключевые данные для тестирования:

* базовые URL для веб‑сайта и API;
* тайм‑ауты и настройки ожидания;
* ID фильмов, сериалов, актёров;
* поисковые запросы для проверки функционала поиска;
* токен для доступа к API.

---

## 8. Класс `DataProvider`

Класс `DataProvider` (файл `DataProvider.py`) обеспечивает доступ к тестовым данным из test_data.json. Он предоставляет методы для получения:
* строковых и целочисленных значений по ключу;
* токена API (из переменных окружения или test_data.json);
* тестовых данных для проверки фильмов, эпизодов, актёров и поиска.

---

## 9. Отчёты Allure

Отчёты Allure генерируются автоматически при запуске тестов с опцией `--alluredir`.
Запустите тесты с генерацией отчёта Allure:
```bash 
pytest --alluredir=./allure-results
```

Эта команда запустит все тесты и сохранит результаты в директорию `./allure-results`

После завершения тестов, чтобы посмотреть отчёт, выполните команду:
```bash
allure serve ./allure-results
```

Эта команда запустит локальный сервер и откроет отчёт в браузере.


По завершении выполнения тестов отчёт Allure содержит детализированную информацию о результатах для каждого шага и тестов в целом.
